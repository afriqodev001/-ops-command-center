Format help

CSV columns (in any order, case-insensitive)

{{ columns|join:", " }}
  • Notification Emails and Suppression Records — semicolon-separated lists.
  • Banner — Yes/No.
  • Notification to Partners for Outage and Suppression — free text guidance, often with conditions. Examples: "PMT is Active/Active, do not send communication. Yes for BCP event." The AI review reads the full text and judges based on the change context.
  • Outage Impact — single text cell. Can be a generic statement (e.g. "Logins blocked during deploy window") OR a per-app split. CSV cannot encode per-app additional emails — use JSON for that.

JSON shape (for full per-app fidelity)

{
  "rows": [
    {
      "application": "Auth Service",
      "ci": "AUTH-SVC-PROD",
      "outage_impact": [
        {
          "app": "Customer Portal",
          "description": "Logins blocked during deploy",
          "additional_emails": ["portal-team@x.com"]
        },
        {
          "app": "Mobile App",
          "description": "App start fails for unauthenticated users"
        }
      ],
      "notify_partners": "Yes — notify within 24h of deploy window",
      "notification_emails": ["app-owners@x.com"],
      "suppression": "Suppress all Auth alerts during the window",
      "suppression_records": ["ALRT-1234", "ALRT-5678"],
      "banner": true,
      "banner_message": "Auth deploy in progress — login may be intermittent"
    },
    {
      "application": "PMT",
      "ci": "PMT-CORE",
      "outage_impact": [
        {"description": "Payment processing degraded during deploy."}
      ],
      "notify_partners": "PMT is Active/Active and we should not send communication. Yes for BCP event.",
      "notification_emails": ["pmt-oncall@x.com"],
      "suppression": "No — Active/Active masks alerts. Yes if BCP event.",
      "suppression_records": [],
      "banner": false
    }
  ]
}

When the email is drafted, recipients = Notification Emails ∪ all additional_emails from impact entries. The body lists every per-app impact line.
